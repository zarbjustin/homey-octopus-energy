## Debt themes

1. **S51/S52/S53 are debt paydown, not optional polish.** The roadmap already orders budget hardening, decomposition and provenance before heavier features (`docs/handover/sprints-50-58-spec.md:185`).
2. **Privacy debt is mostly in keys, not values.** Value redaction is good (`lib/OctopusMeterDevice.ts:625`), but persisted maps still reveal account/meter-derived keys.
3. **Performance debt follows duplication.** The same REST data is fetched by multiple reporting paths because there is no service-level request cache yet.

⚠ Cross-discipline note: An architect may argue that some S51/S52 items are “refactor-only” and should wait behind user-visible work. The evidence says otherwise: rate budget, repair propagation and per-source freshness directly affect reliability and trust (`ROADMAP.md:76`, `ROADMAP.md:78`).
